Compared against: current live MariaDB database `musicshop` on 2026-03-25

Summary
The overall database model in the documentation is still broadly correct:
- the database is still named `musicshop`
- it still uses the same 6 main tables: `users`, `products`, `favorites`, `orders`, `order_items`, `news`
- the main relationships described in the PDF are still present
- the unique constraints and foreign keys described for `users`, `favorites`, `orders`, and `order_items` still exist
- the `products` indexes described in the PDF still exist, including the full-text index on `name`, `artist`, `description`

Main differences between the PDF and the current setup

1. The `news` table is no longer as simple as described in the PDF.
The PDF says that `news` contains only:
- id
- title
- content
- created_at

The current live table `news` contains:
- id
- title
- subtitle
- content
- image_url
- created_at

This is the biggest schema difference in the documented DB section.

2. The documentation says the PDF reflects the current state through `musicshop.sql`, but the live DB has moved beyond the documented state.
The PDF explicitly says that `musicshop.sql` reflects the current state of the project database.
That is no longer fully true for the running database, because the live DB now includes later changes such as:
- `news.subtitle`
- `news.image_url`
- additional inserted news records
- a larger current product catalog

3. The current live data volume is larger than what the documentation implies.
Current live row counts:
- users: 3
- products: 39
- favorites: 7
- orders: 3
- order_items: 6
- news: 13

The PDF describes the tables correctly at a conceptual level, but it reflects an earlier content snapshot and does not mention the expanded current data volume.

4. The current `products` table remains structurally aligned with the documentation, but the live catalog is richer than the examples in the PDF.
The PDF describes `products` as a unified table for instruments, music, merch, and accessories, with fields such as:
- format
- music_genre
- size
- color
- merch_type
- instrument_subtype
- accessory_type

That is still correct.
However, the live DB now contains 39 products and a wider catalog than the examples described in the document.
The documented explanation is still valid, but it is no longer a full picture of the current dataset.

5. The current live DB appears to be closer to the newer dump (`musicshop_last.sql`) than to the state described in the PDF text.
The PDF database discussion was written around the older documented snapshot.
The current running database includes later catalog/news changes that are better reflected by the newer project state than by the prose in the PDF.

Recommended documentation update
The DB section in the PDF should be updated so that the `news` table description matches the current schema:
- add `subtitle`
- add `image_url`

It would also be more accurate to replace the statement that `musicshop.sql` reflects the current state with wording that points either to:
- the current live database state, or
- the newest maintained SQL dump used as the project reference
