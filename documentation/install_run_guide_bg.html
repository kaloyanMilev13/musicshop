MusicShop
Инструкции за инсталиране и стартиране

Този документ описва как проектът MusicShop да бъде прехвърлен с флашка и стартиран на друг компютър. Проектът е уеб приложение с PHP + Apache + MariaDB/MySQL.

1. Какво трябва да има на флашката

Най-сигурният вариант е да се копира цялата папка musicshop без преименуване.

Ако се подготвя минимален пакет, задължително трябва да присъстват:

  • всички основни .html файлове в корена на проекта;
  • папка api/;
  • папка css/;
  • папка js/;
  • папка images/;
  • файл musicshop_last.sql - основният SQL архив за базата данни.

Важно: папката трябва да се казва точно musicshop. В кода има заявки от вида /musicshop/api/.... Ако папката се преименува, сайтът няма да може да зарежда данни от API-то.

Незадължителни файлове:

  • start_stack.py и stop_stack.py - помощни скриптове за Linux;
  • reset_db_kala.sh - помощен Linux скрипт за нулиране на базата;
  • папка documentation/ - документация, не е нужна за работата на сайта;
  • старите SQL файлове musicshop.sql, seed_imaginary_news_20260325.sql, product_description_updates_20260326.sql - не са нужни, ако се използва musicshop_last.sql.

2. Необходим софтуер

За стартиране на проекта на Windows най-лесно е да се използва XAMPP.

  • Apache
  • MariaDB / MySQL
  • PHP
  • phpMyAdmin

Ако компютърът няма интернет, може на флашката да се добави и инсталатор за XAMPP. Ако XAMPP вече е инсталиран, не е нужно да се копира.

3. Инсталиране и стартиране на Windows чрез XAMPP

3.1. Копиране на проекта

  1. Инсталирайте XAMPP.
  2. Отворете папката C:\xampp\htdocs\.
  3. Копирайте в нея цялата папка musicshop от флашката.
  4. Крайният път трябва да бъде C:\xampp\htdocs\musicshop\.

3.2. Стартиране на сървърите

  1. Отворете XAMPP Control Panel.
  2. Стартирайте Apache.
  3. Стартирайте MySQL.

3.3. Създаване на базата данни

  1. Отворете http://localhost/phpmyadmin.
  2. Създайте нова база данни с име musicshop и кодировка utf8mb4_unicode_ci.
  3. Отворете базата musicshop.
  4. Изберете таба Import.
  5. Импортирайте файла musicshop_last.sql.

3.4. Потребител за базата данни

Във файла api/db.php са зададени следните настройки:

  • Host: 127.0.0.1
  • Port: 3306
  • User: kala
  • Password: kala
  • Database: musicshop

Има два възможни начина:

  1. Да се създаде MySQL потребител kala с парола kala.
  2. Или да се редактира файлът api/db.php и да се заменят данните с локалните настройки на MySQL/XAMPP, например потребител root с празна парола.

Примерни настройки за XAMPP:

  • $DB_USER = 'root';
  • $DB_PASS = '';

3.5. Стартиране на сайта

След успешен импорт и работещи Apache/MySQL, сайтът се отваря от:

http://localhost/musicshop/

4. Проверка дали проектът работи

  1. Отворете началната страница.
  2. Проверете дали се зареждат продукти и новини.
  3. Тествайте регистрация на нов потребител.
  4. Тествайте вход в системата.
  5. При нужда проверете връзката към базата през http://localhost/musicshop/api/test_db.php.

5. По желание: стартиране на Linux

Проектът първоначално е подготвен за среда с Apache (httpd) и MariaDB.

  1. Копирайте проекта в уеб директория, така че да се достъпва като /musicshop.
  2. Стартирайте Apache и MariaDB.
  3. Създайте база musicshop и импортнете musicshop_last.sql.
  4. Настройте потребителските данни във файла api/db.php.

Файловете start_stack.py и stop_stack.py работят само ако на Linux има услуги с имена httpd и mariadb и ако потребителят има права да ги стартира.

6. Чести проблеми

  • Празни страници или липсващи продукти: най-често има проблем с базата данни или импортът не е направен.
  • Грешка при свързване към базата: проверете настройките във файла api/db.php.
  • 404 / API not found: проектът вероятно не е в папка musicshop.
  • Липсващи изображения: уверете се, че папка images/ е копирана изцяло.

7. Препоръка за предаване с флашка

Препоръчителна структура на флашката:

  • musicshop/ - целият проект;
  • XAMPP Installer - по желание, ако ще се инсталира офлайн;
  • този файл с инструкции.

Ако има достатъчно място, най-практично е да се даде цялата папка на проекта, а не само минимален подбор от файлове. Така рискът да липсва важен ресурс е най-малък.
